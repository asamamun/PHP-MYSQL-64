PHP Error Handling Overview

Types of Error in PHP

  • Parse Errors: Syntax errors.
  • Fatal Errors: Occur when PHP can't run the script.
  • Warning: Script continues, but somethingâ€™s wrong.
  • Notice: Minor issues, like undefined variables.
  • Deprecated: Features that will be removed in future versions.

error_reporting Directive

Controls which types of errors are reported. Example:

error_reporting(E_ALL);

PHP's Error-Reporting Levels

  • E_ERROR - Fatal run-time errors
  • E_WARNING - Run-time warnings (non-fatal)
  • E_PARSE - Compile-time parse errors
  • E_NOTICE - Run-time notices
  • E_ALL - All errors and warnings

display_errors Directive

Determines if errors should be displayed in the browser.

display_errors = On

log_errors Directive

Determines whether script errors should be logged.

log_errors = On

error_log Directive

Specifies the file where script errors should be logged.

error_log = /path/to/php-error.log

Exception Handling in PHP

PHP allows you to handle exceptions using try, catch, and finally blocks.

Exception Class in PHP

All exceptions must be derived from the Exception class. You can extend it to create custom exceptions.

try-catch-finally Block

      try {
          // Code that may throw an exception
      } catch (Exception $e) {
          // Handle exception
      } finally {
          // Code that always runs
      }
