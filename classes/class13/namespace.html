PHP Namespace Example with Class and Method Conflict

1. Without Namespace (Conflict)

In this example, we have two classes with the same name `Car`, each in different files. Both contain a method named `drive`.

File 1: Car.php
<?php
class Car {
  public function drive() {
    echo "Driving car from File 1";
  }
}
?>
File 2: Car.php
<?php
class Car {
  public function drive() {
    echo "Driving car from File 2";
  }
}
?>

If you try to use these classes in the same script, you'll get a fatal error due to the conflict in class names.

2. Solution Using Namespaces

To resolve the conflict, you can use namespaces. Each class will be placed in a different namespace, ensuring they do not conflict.

File 1: NamespaceOne/Car.php
<?php
namespace NamespaceOne;

class Car {
  public function drive() {
    echo "Driving car from NamespaceOne";
  }
}
?>
File 2: NamespaceTwo/Car.php
<?php
namespace NamespaceTwo;

class Car {
  public function drive() {
    echo "Driving car from NamespaceTwo";
  }
}
?>

Now, each class is in its own namespace, and you can use them in the same script without conflicts.

3. Using the Classes

Here's how you can use both classes in a single script:

Using the Namespaced Classes
<?php
// Include the files
require_once 'NamespaceOne/Car.php';
require_once 'NamespaceTwo/Car.php';

// Use the classes with their namespaces
use NamespaceOne\Car as CarOne;
use NamespaceTwo\Car as CarTwo;

// Instantiate objects from each namespace
$carOne = new CarOne();
$carTwo = new CarTwo();

// Call the drive method from each
$carOne->drive(); // Output: Driving car from NamespaceOne
echo "<br>";
$carTwo->drive(); // Output: Driving car from NamespaceTwo
?>

This solution shows how namespaces can help avoid naming conflicts, making your codebase cleaner and more maintainable.
