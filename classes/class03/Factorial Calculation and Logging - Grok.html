Lets break down how these three files (factorial-log.php, find-factorial.php, and result.php) work together to handle value passing, logging, and displaying results. Ill explain the flow of data step-by-step, focusing on how data is sent from HTML to PHP, processed on the server, logged, and then displayed.

1. factorial-log.php (HTML Form)

This file contains an HTML form that collects user input and sends it to the server.

Code Breakdown:

html
<form action="find-factorial.php" method="get"> <input type="text" name="studentname" placeholder="Enter your name" required> <input type="number" name="n" placeholder="Enter a number to find its factorial" required min="2"> <input type="submit" value="Submit" name="findFactorial"> </form>

How It Works:

  • Form Action and Method:
    • action="find-factorial.php": When the form is submitted, the data is sent to find-factorial.php.
    • method="get": The form data is sent via the URL as query parameters (e.g., find-factorial.php?studentname=John&n=5&findFactorial=Submit).
  • Inputs:
    • <input type="text" name="studentname">: Collects the users name. The name attribute (studentname) becomes the key in the query string.
    • <input type="number" name="n">: Collects a number for factorial calculation. The min="2" ensures the number is at least 2, and required makes it mandatory.
    • <input type="submit" name="findFactorial">: The submit button. Its name attribute (findFactorial) is included in the query string when clicked.
  • Data Sent to Server: When the user enters, say, "Alice" as the name and "5" as the number, then clicks "Submit," the browser sends a GET request like this:
    text
    http://example.com/find-factorial.php?studentname=Alice&n=5&findFactorial=Submit
    This URL-encoded data is accessible in find-factorial.php via the $_GET superglobal.

2. find-factorial.php (Server-Side Processing and Logging)

This PHP file processes the form data, calculates the factorial, logs the request, and redirects to result.php.

Code Breakdown:

php
<?php if(isset($_GET['findFactorial'])) { $name = $_GET['studentname'] ?? "ANON"; $n = $_GET['n'] ?? 1;//null coalescing operator ?? //log the search in a file $fh = fopen("factorial-log.txt", "a"); fwrite($fh, "$name: searched for Factorial of $n at " . date('Y-m-d H:i:s a') . "\n"); fclose($fh); $f = 1; for ($i=1; $i <= $n ; $i++) { $f = $f * $i; } $message = "Hey $name, Factorial of $n = " . $f; header("Location: result.php?message=$message"); } ?>

How It Works:

  • Checking Form Submission:
    • if(isset($_GET['findFactorial'])): Ensures the code runs only if the form was submitted (i.e., findFactorial is in the query string).
  • Retrieving Data:
    • $name = $_GET['studentname'] ?? "ANON": Uses the null coalescing operator (??) to get the studentname from the URL. If its not set, defaults to "ANON".
    • $n = $_GET['n'] ?? 1: Gets the number n from the URL, defaulting to 1 if not provided.
  • Logging:
    • fopen("factorial-log.txt", "a"): Opens factorial-log.txt in append mode ("a") to add new entries without overwriting existing ones.
    • fwrite($fh, "$name: searched for Factorial of $n at " . date('Y-m-d H:i:s a') . "\n"): Writes a log entry like "Alice: searched for Factorial of 5 at 2025-03-21 10:30:45 am".
    • fclose($fh): Closes the file handle.
  • Factorial Calculation:
    • $f = 1: Initializes the factorial result.
    • for ($i=1; $i <= $n ; $i++) { $f = $f * $i; }: Calculates the factorial (e.g., for n=5, it computes 1 * 1 * 2 * 3 * 4 * 5 = 120).
  • Preparing the Result:
    • $message = "Hey $name, Factorial of $n = " . $f: Creates a string like "Hey Alice, Factorial of 5 = 120".
  • Redirecting:
    • header("Location: result.php?message=$message"): Redirects the browser to result.php, passing the $message as a query parameter. The URL becomes:
      text
      http://example.com/result.php?message=Hey+Alice,+Factorial+of+5+=+120
    • Note: Spaces in $message are URL-encoded as + or %20.

3. result.php (Displaying Results)

This file receives the message from find-factorial.php and displays it to the user.

Code Breakdown:

php
<?php if(isset($_GET['message'])) { $message = $_GET['message']; echo <<<ALERT <div class="alert alert-success alert-dismissible fade show" role="alert"> $message . <a href="factorial-log.php">Find Factorial</a> <button type="button" class="btn-close" data-bs-dismiss="alert" aria-label="Close"></button> </div> ALERT; } ?> <a href="factorial-log.php" class="btn btn-outline-primary">Factorial</a>

How It Works:

  • Retrieving the Message:
    • if(isset($_GET['message'])): Checks if a message parameter exists in the URL (from the redirect).
    • $message = $_GET['message']: Assigns the query parameter (e.g., "Hey Alice, Factorial of 5 = 120") to $message.
  • Displaying the Result:
    • echo <<<ALERT ... ALERT;: Uses a heredoc to output an HTML alert box (Bootstrap-styled).
    • Inside the alert:
      • $message: Displays the factorial result (e.g., "Hey Alice, Factorial of 5 = 120").
      • <a href="factorial-log.php">Find Factorial</a>: Adds a link back to the form.
      • A close button allows the user to dismiss the alert.
  • Additional Link:
    • <a href="factorial-log.php" class="btn btn-outline-primary">Factorial</a>: A button outside the alert links back to the form page.
  • Output Example: If the URL is result.php?message=Hey+Alice,+Factorial+of+5+=+120, the page shows:
    text
    [Alert Box: "Hey Alice, Factorial of 5 = 120. Find Factorial" with a close button] [Button: "Factorial"]

Overall Data Flow:

  1. User Input (Client to Server):
    • The user fills out the form in factorial-log.php (e.g., name="Alice", n=5) and submits it.
    • The browser sends a GET request to find-factorial.php with the data in the URL.
  2. Server Processing and Logging:
    • find-factorial.php retrieves the data (studentname and n) from $_GET.
    • It logs the request to factorial-log.txt.
    • It calculates the factorial and constructs a message.
    • It redirects to result.php, passing the message in the URL.
  3. Result Display (Server to Client):
    • result.php retrieves the message from $_GET and displays it in an alert box.
    • It provides a link/button to return to the form.

Key Points:

  • Value Passing: Data moves from the HTML form to PHP via the $_GET array (URL query parameters) and between PHP files via URL redirects.
  • Logging: The log is stored in factorial-log.txt with timestamps, tracking every request.
  • Result Display: The final output is rendered as HTML with Bootstrap styling for a polished look.

This system is simple but effective for demonstrating form handling, server-side processing, and result display in PHP! Let me know if youd like further clarification.
